Day 2. Событие MouseHover

16/11/2023

Немного усложним задачу предыдущего примера. Добавим для объекта Label1 обработку еще одного события – MouseHover. Событие MouseHover наступает, когда пользователь курсором мыши “зависает” над каким-либо объектом (от англ. hover – реять, парить), а не просто проводит мышью над объектом. Можно сказать также, что событие MouseHover происходит, когда курсор мыши наведен на элемент.

Итак, в нашем примере программа должна содержать на экранной форме метку Label и кнопку Button. Метка будет отображать текст “Microsoft Visual C#”, а при клике на кнопку, на которой написано “Нажми меня”, откроется диалоговое окно с сообщением “Всем привет!”. Кроме того, когда курсор мыши окажется наведен на метку (то самое событие MouseHover), должно открыться диалоговое окно с текстом “Событие Hover”.

Для выполнения этой задачи запускаем VS 2022, щелкаем по Create a new project. В открывшемся окне New Project выбираем пункт Visual C#, а потом в области Templates (Шаблоны) – шаблон Windows Forms App. В качестве имени проекта введем имя Hover и щелкнем на кнопке OK.

В дизайнере формы из панели Toolbox перетащим на форму метку label и кнопку Button, а затем немного уменьшим размеры формы на свой вкус. Теперь добавим три обработчика событий в код. Для этого в панели Properties щелкаем на значке молнии (Events) и двойными щелчками последовательно выбираем:

  • событие загрузки формы Load;
  • событие клика по кнопке button1 – Click;
  • событие “зависания” курсора мыши на метке label1 – MouseHover.

При этом осуществится переход на вкладку кода Form1.cs и VS 2022 создаст три пустых обработчика события. Например, обработчик события MouseHover будет иметь вид:

private void label1_MouseHover(object sender, EventArgs e)
{

}

Между этими двумя фигурными скобками напишем вызов диалогового окна:

MessageBox.Show("Событие Hover!");

Теперь проверим возможности программы – нажимаем клавишу F5, “зависаем” курсором мыши над Label1, щелкаем по кнопке Button1. Все работает.

Итак, вернемся к событию загрузки формы в оперативную память при старте программы – Form1_Load – для него VS 2022 создала пустой обработчик:

private void Form1_Load(object sender, EventArgs e)
{

}

Здесь, между двумя этими фигурными скобками, обычно вставляют свойства различных объектов и даже часто пишут много строчек программного кода. Давайте назначим свойству Text объекта label1 значение Microsoft Visual C#:

label1.Text = "Microsoft Visual C#";

Аналогично для объекта button1:

button1.Text = "Нажми меня";

Далее изменим название формы (т.н. строка заголовка) с Form1 на что-нибудь осмысленное, например “Приветствие”. Для этого присваиваем эту строку свойству Text формы. Поскольку мы изменяем свойство объекта Form1 внутри подпрограммы обработки события, связанного с формой, следует к форме обращаться или через ссылку this, или используя ключевое слов base:

base.Text = "Приветствие";

Написав последнюю строку кода, мы должны увидеть на экране код, представленный ниже:

namespace MouseHover
{
    public partial class Form1 : Form
    {
        public Form1()
        {
            InitializeComponent();
        }

        // Процедура обработки события загрузки формы
        private void Form1_Load(object sender, EventArgs e)
        {
            label1.Text = "Microsoft Visual C#";
            button1.Text = "Нажми меня";
            base.Text = "Приветствие";
        }

        private void button1_Click(object sender, EventArgs e)
        {
            MessageBox.Show("Всем привет!");
        }

        private void label1_MouseHover(object sender, EventArgs e)
        {
            MessageBox.Show("Событие Hover!");
        }
    }
}

Комментарии, поясняющие работу программы, в окне редактора кода выделены зеленым цветом, чтобы в тексте он выразительно отделялся от прочих элементов программы. Эти комментарии пишут после двух слэшей (//) или внутри пар /* */.

Исходный код этого примера можно посмотреть, открыв решение Hover.sln в папке Hover.
