#9. Свойство Visible и всплывающая подсказка ToolTip

23/11/2023

Рассмотрим возможности свойства Visible. Программа пишет в метку Label некоторый текст, а пользователь с помощью кнопки делает этот текст невидимым, а затем опять видимым и т.д. При зависании мыши над кнопкой появляется всплывающая подсказка “Нажми меня”.

Для выполнения этой задачи запустим VS 2022, выбираем проект Windows Forms App и называем его Balloon. Из панели элементов управления Toolbox в форму перетаскиваем метку Label, кнопку Button и всплывающую подсказку ToolTip. Только в этом случае каждый элемент управления в форме (включая форму) получает свойство ToolTip on Tip. Убедиться в этом можно, посмотрев свойства (окно Properties) элементов управления.

Для кнопки Button1 у свойства ToolTip on Tip мы могли бы написать Нажми меня. Однако я предлагаю написать это непосредственно в программном коде. В таком случае программисту не придется долго искать соответствующее свойство, когда он станет применять этот фрагмент в своей следующей программе.

Перейдем на вкладку программного кода, нажав клавишу F7, и напишем следующий код:

// Программа пишет в метку Label некоторый текст, а пользователь
// с помощью кнопки делает этот текст либо видимым, либо
// невидимым. Здесь используется свойство Visible. При зависании мыши
// над кнопкой появляется подсказка "Переключатель видимости"
namespace PopupHint
{
    public partial class Form1 : Form
    {
        public Form1()
        {
            InitializeComponent();
        }

        private void Form1_Load(object sender, EventArgs e)
        {
            // Обрабатываем событие "загрузка формы"
            this.Text = "Житейская мудрость";
            label1.Text =
                "Сколько ребенка ни учи хорошим манерам," + "\n" +
                "он будет поступать так, как папа с мамой";
            // label1.TextAlingn = ContentAlignment.MiddleCenter;
            button1.Text = "Кнопка";

            // Создаем элемент управления toolTip и задаем ему событие
            ToolTip toolTip1 = new ToolTip();
            toolTip1.Active = true;
            toolTip1.SetToolTip(this.button1, "Переключатель" + "\n" + "видимости");
        }

        private void button1_Click(object sender, EventArgs e)
        {
            // Обработка события "щелчок на кнопке":

            // Можно программировать так:
            // if (label1.Visible == true) label1.Visible = false;
            // else label1.Visible = true;
            // или так:
            // label1.Visible = label1.Visible ^ true;

            // Или самый простой вариант:
            label1.Visible = !label1.Visible;
        }

    }
}

При обработке события загрузки формы свойству Text метки присваиваем некоторый текст, “склеивая” его с помощью знака “плюс” (+) из отдельных фрагментов. Использование в текстовой строке символов “\n” означает перенос текста на новую строку. Можно перенести на новую строку и с помощью строки Environment.NewLine. В перечислении Environment можно выбрать и другие управляющие символы. Свойство метки TextAlign располагает текст метки по центру и посередине (MiddleCenter). Выражение, содержащее ToolTip1, устанавливает (Set) текст всплывающей подсказки для кнопки button1 при “зависании” над ней указателя мыши. Для всплывающей подсказки используется стандартное прямоугольное окно.

Чтобы в программном коде получить пустой обработчик события “щелчок мышью на кнопке”, следует в дизайнере формы (т.е. на вкладке Form.cs[Designer]) двойным щелчком щелкнуть по кнопке Button1. В теле обработчика этого события видим 5 закомментированных строчек, в которых записана логика включения/выключения видимости метки. Логика абсолютно понятна: если свойство видимости (Visible) включено (true), то его следует выключить (false): иначе (else) – включить.

Да, несколько путано, но разобраться можно. И все работает. Проверяем. Кнопку можно нажимать мышью, клавишей Enter и клавишей “пробел”.

Однако можно пойти другим путем. Именно поэтому пять строчек этой логики переведены в комментарий. Мы уже встречались с побитовым оператором (^) (исключающее ИЛИ). Напоминаю, что этот оператор выбирает “да” (true), сравнивая “нет” и “да”, и выбирает “нет” (false), сравнивая “да” и “да”. Однако можно еще больше упростить написание кода:

label1.Visible = !label1.Visible;

То есть при очередной передаче управления на эту строчку свойство Label1.Visible будет принимать противоположеное значение. Мы убедились, что можно по-разному программировать подобные ситуации.

Совет. Очень удобно комментировать несколько строк кода в редакторе VS 2022 сразу – вначале выделяя их, а затем применяя комбинацию клавиш Ctrl + K | C (Comment). Чтобы убрать с нескольких строк знак комментария, можно – вначале выделить их, а затем воспользоваться уже другой комбинацией клавиш: Ctrl + K | U (Uncomment).

Исходный код этого примера можно посмотреть, открыв решение PopupHint.sln в папке PopupHint.
