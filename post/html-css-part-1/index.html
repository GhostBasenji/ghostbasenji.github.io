HTML5 и CSS3. Часть 1: Пишем наш первый код

25/04/2023

Начинаем учиться работать с HTML и CSS, не заблудившись по пути.

Основа

Давайте начнем с самого начала. HTML (Hyper-Text Markup Language, язык гипертекстовой разметки) технически не является языком программирования. Вместо этого он начинался как способ отображения текста, изображений и других типов данных. Со временем был разработан и дополнен стандарт, чтобы расширить возможности HTML. Эти изменения привели к серьезному пересмотру спецификации HTML. Последней спецификацией, которая была выпущена консорциумом W3C, является HTML5.

А теперь пройдемся по основам CSS, а потом сразу перейдем к делу.

CSS (Cascading Style Sheets, Каскадные таблицы стилей) – это способ изменять внешний вид HTML-страницы. На самом деле, CSS необязательно применять только к HTML, но это, безусловно, его основное применение. Так же, как и HTML, CSS прошел через множество итераций. Также, как и спецификация HTML, спецификация CSS поддерживается W3C.

CSS был разработан для того, чтобы отделить дизайн от контента. Контент помещается в HTML, а CSS придает этому контенту привлекательный вид. Благодаря такому разделению мы можем легко изменять внешний вид своего сайта без необходимости изменять структуру или контент.

Итак, поехали

Вот и все, мы закончили с введением и теперь готовы занять руки. Итак, наш план прогресса таков:

  1. Создание базовой веб-страницы Hello World.
  2. Деконструкция веб-страницы для идентификации все ее части.

Это послужит нам началом. Оттуда мы к разметке HTML, а затем добавим наш первый CSS и посмотрим, как все это начинается что-то меняться. Среда разработки – это очень личная вещь. У каждого человека есть свое собственное представление о том, какая IDE лучше. HTML и CSS не нуждаются в компиляции (как это делают C# и другие языки). Это действительно открывает перед нами возможности выбора редактора, который мы можем использовать. Это означает, что даже Notepad является допустимым редактором для HTML и CSS. В этом уроке будем использовать VS Code от Microsoft.

Создание страницы Hello World

Готовы создать нашу первую веб-страницу? Давай сделаем это. Запускаем VS Code. Создаем новый файл через главное меню File > New File… В появившемся окошке создания файла напишем index.html.

Мы выбрали имя index.html потому что веб-серверы ищут определенные имена файлов в качестве точки входа на веб-сайт. Как правило, они ищут index.html или default.html. Так что, стандартное имя точки входа - index.html. Когда мы называем страницу index.html, эта страница станет страницей по умолчанию, которая будет загружена, если явно не указана ни одна страница. Это означает, что index.html будет нашей домашней страницей.

После того, как мы создали нашу страницу, по умолчанию она содержит следующий HTML-код:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
   
</body>
</html>

Это базовый каркас веб-страницы. Мы потом вернемся и проанализируем, что означает каждая строка, но давайте сначала сосредоточимся на одном разделе: body. HTML состоит в основном из “тегов”. Большинство тегов имеют открывающий и закрывающий теги. Например, – это тег. Его закрывающий тег – . Обращаем внимание на слэш перед body. Всякий раз, когда видим этот слэш перед именем тега, то мы знаем, что это закрывающий тег. Когда мы сталкиваемся с парой тегов, подобной body, все, что находится между открывающим и закрывающим тегами, находится внутри этого тега. Это означает, что если поместим что-то между тегами body, это что-то будет в теле нашей страницы.

В нашем случае мы хотим стать веб-разработчиками, поэтому давайте поместим что-нибудь в тело нашей HTML-страницы. Введем “Hello World!” (без кавычек) между открывающим и закрывающим тегами body, например так:

<body>
   Hello World
</body>

Вот и все. Теперь сохраним страницу, а потом запустим ее, нажав Ctrl+F5. Перед нами появится страница:

Не очень зрелищно, но это только начало. Поздравляем, мы только что создали нашу первую веб-страницу.

Анализ базовой веб-страницы

Теперь вернемся назад и рассмотрим эту HTML-разметку более подробно:

<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="UTF-8">
    	<meta name="viewport" content="width=device-width, initial-scale=1.0">
    	<title>Document</title>
</head>
<body>
</body>
</html>

Первая строка – это специальная директива, называемая DocType. Эта директива сообщает браузере, на основе какой спецификации HTML была создана эта страница. Как уже ранее говорилось, мы работаем с HTML5. По мере изменения спецификации менялись и теги, и их использование. Чтобы новая спецификация не портила существующие страницы, был использован DocType для определения того, какая спецификация использовалась для создания страницы. На самом деле DocType больше не используется для этого. Теперь мы включаем его просто для того, чтобы указать, что это страница соответствует последним стандартам. Если не укажем DocType, то страница будет некорректно показываться в браузере, потому что браузер использует самую старую спецификацию, какая есть у него для отображения нашей страницы. Суть здесь в том, что нам необходимо включать эту строку, но вообще не нужно ее изменять.

Далее идет тег html. Этот тег интересен тем, что также знакомимся с атрибутом. Атрибут каким-либо образом расширяет тег или работает с ним. В нашем случае этот тег указывает, что основным языком для всего, что находится между тегами html, является английский. Сам тег html обертывает весь документ целиком. У нас не должно быть никаких тегов (кроме DocType) вне тегов html.

Потом идет тег head. Воспримем раздел head как заголовок электронного письма. Он содержит всю информацию о странице. Большинство элементов в разделе head необязательны, но заголовок обязателен. Это поднимает еще один момент, который мы начинаем понимать: теги могут быть вложены друг в друга. Единственное, за чем нужно следить, – это за тем, чтобы мы закрывали свои теги в правильном порядке. Например, нам нужно закрыть тег title прежде, чем закроем тег head.

Следующий тег, который мы видим, – это мета-тег. Это интересно по нескольким причинам. Во-первых, у него есть атрибут (кодировка). Далее, у него нет закрывающего тега. Давайте начнем с самого начала. Мета-теги обычно представляют собой информацию о странице. Эти теги не видны пользователю. Вместо этого они используются браузерами, веб-серверами и другими машинными считывателями для получения информации о странице. В нашем случае мы сообщаем браузеру, что эта страница использует utf-8 (Unicode) в качестве набора символов. Это широко используемый набор символов, который может работать с большинством языков. Другие мета-теги могут содержать описание сайта, автора и т.д. По умолчанию он тут включен.

Далее видим тег viewport. Он отвечает за область на экране устройства, которая видна пользователю без прокрутки экрана. Представьте страницу, которая нормально отображается на экране ноутбука. Буквы, значки и прочие элементы находятся в оптимальном размере, удобно читать текст. Если не задать тег viewport, то наша страница на экране смартфона окажется совсем неудобной для чтения. Читателю придется двигать изображения и текст влево и вправо, вниз и вверх. Первый параметр – Width – дает команду браузеру адаптировать по ширине устройства страницу. Масштабирование ширины страницы определяется автоматически, если указано значение “device-width”. Также можно установить значение атрибута от 200 до 1000 пикселей. Следующий параметр – initial-scale – означает коэффициент масштабирования. Если указано значение “1”, разрешение CSS и устройства приравниваются друг к другу независимо от размера экрана. Можно указать любое значение параметра от 0.1 до 10.

Теперь у мета-тега нет закрывающего тега. Но все равно этот тег самозакрывается, так как в случае с мета-тегами никогда не должно быть истинного закрывающего тега. В HTML5 технически поддерживается отсутствие закрывающего тега для некоторых тегов. Однако раньше, когда элемент не имел закрывающего тега, его обозначали слэшом в конце. HTML5 прекрасно работает с элементами, которые имеют слэш в конце. Это означает, что обычно это предпочтительный метод обработки тегов, у которых нет закрывающих тегов.

Раздел title внутри тега head – это то место, где мы размещаем заголовок страницы. Title – это то, что отображается в строке заголовка или заголовке вкладки в веб-браузере. Например, если модифицируем наш title следующим образом:

<title>Наш первый веб-сайт</title>

Наша веб-страница теперь будет выглядеть так (обращаем внимание на разницу в названии вкладки вверху):

На этом мы завершаем наш обзор разметки по умолчанию. Подводим итог – HTML состоит из тегов. Большинство тегов имеют открывающий и закрывающий теги. Вложенные теги должны быть закрыты прежде, чем можно будет закрыть их родительский тег. Два основные раздела (или секции) – это head и body. Данные о странице обычно размещают в head, а данные страницы – в body.

Другие HTML-теги

Так как это вводная статья, поэтому не будем углубляться в HTML-теги. Однако, когда мы только начинаем, есть несколько тегов, которые мы должны знать, как их использовать.

Теги заголовков

Теги headers (теги заголовков) позволяют определить структуру нашей веб-страницы. Всего существует шесть различных тегов заголовков (от h1 до h6). Каждый из них указывает уровень заголовка (h1 – самый высокий и, как правило, самый большой, а h6 – самый маленький и, как правило, наименьший). Чтобы продемонстрировать это, давайте изменим нашу страницу, чтобы иметь общий заголовок и пару заголовков разделов, например:

<body>
    <h1>Простая домашняя страница</h1>
    <h2>Раздел 1</h2>
    Hello World
    <h2>Раздел 2</h2>
</body>

Так как “Hello World” – это просто контент, а не заголовок, поэтому оставляем его вне тега header. Итак, все это разбивает документ на разделы следующим образом:

Основы CSS

Теперь давайте посмотрим, зачем надо использовать CSS. Прямо сейчас наша веб-страница очень проста. Это черно-белое и все такое прочее. CSS - это то, что мы меняем все это, в том числе и внешний вид веб-страницы. Начнем с добавления нового файла. На этот раз создадим файл с именем styles.css.

После того, как создали новый файл CSS, теперь напишем следующий код (файл по умолчанию начинается с этих слов):

body {
}

Сейчас самое подходящее время для обсудить, что делает CSS. CSS каким-то образом изменяет части нашего HTML. Например, он может изменить шрифт определенного раздела, изменить цвет фона другого раздела или добавить дополнительное пространство вокруг определенных тегов. Итак, это означает, что правило CSS состоит из двух частей: затрагиваемый элемент и то, что хотим изменить в этом элементе. Глядя на вышеприведенный CSS, увидим body. Это выглядит знакомо. Это правило CSS влияет на текст нашей HTML-страницы. Теперь обычно у нас было бы что-то внутри фигурных скобок (то, что нужно изменить), но на данный момент наше правило пусто. Почему бы нам не изменить это, а потом посмотреть, что произойдет. Давай изменим шрифт для основной части (и всего, что внутри нее) на “Comic Sans MS”:

body {
   font-family: 'Comic Sans MS';
}

При обновлении страницы ничего не меняется. Почему? Вспоминаем, что когда веб-сервер запускает сайт, он запускает index.html по умолчанию. Это то, что он делает (это то, что хотим). Однако он не знает, нужно ли включать наш CSS-файл. Нам нужно добавить наш CSS-файл в качестве ресурса в нашу страницу index. Мы делаем это, связываясь с ним следующим образом:

<head>
    <meta charset="utf-8" />
    <title>Наш первый веб-сайт</title>
    <link rel="stylesheet" type="text/css" href="styles.css" />
</head>

Обращаем внимание на тег link на четвертой строке. Именно здесь регистрируем наш CSS-файл. Итак, у этого тега есть несколько атрибутов. Первый атрибут – rel – определяет взаимосвязь между двумя документами. В этом случае говорим, что styles.css – это таблица стилей, которую мы используем в этой веб-странице. Далее следует тип документов. Мы указываем “text/css”, чтобы указать, что это документ CSS. Наконец, href – это путь к нашему CSS-файлу. В нашем случае, поскольку два файла находятся на одном уровне (то есть находятся в одной папке), мы можем просто указать имя файла, не указывая путь. Это также может быть полный URL-адрес, указывающий на внешний сайт. Например, если наша таблица стилей хранилась где-то еще в Интернете.

Итак, это основы CSS. Мы определяем, какой тег задействовать, а потом внутри фигурных скобок указываем, что нужно изменить в шаблоне “элемент - двоеточие - значение” (пример: font-family: ‘Comic Sans MS’). Однако это поднимает вопрос о том, что мы бы сделали, если захотим изменить только один раздел. Например, что, если мы захотим внести изменения в абзац “Hello World!”, но не трогая абзац “Как твои дела?”? Вот тут и вступают в игру атрибуты. В HTML можно добавить атрибут id к любому тегу, чтобы однозначно идентифицировать его, например:

<p id="intro">Hello World</p>
<p>Как твои дела?</p>

Затем мы можем настроить таргетинг только на этот первый абзац, используя CSS следующим образом:

#intro {
   font-weight: bold;
}

Обращаем внимание на то, что знак # указывает на тег id. Теперь запустим страницу, чтобы увидеть изменения:

Теги id хороши, но у них есть недостаток: их можно использовать только один раз. У нас не может быть двух тегов с одинаковым атрибутом id. Именно атрибут class значительно упрощает задачу. Атрибут class можно размещать более чем в одном теге. Например, если бы нам нужно добавить третий абзац в нашу страницу, мы могли бы добавить атрибуты class к последним двум абзацам следующим образом:

<body>
    <h1>Простая домашняя страница</h1>
    <h2>Раздел 1</h2>
    <p id="intro">Hello World!</p>
    <p class="sub-paragraph">Как твои дела?</p>
    <p class="sub-paragraph">Очень хорошо.</p>
    <h2>Раздел 2</h2>
</body>

Обращаем внимание, что последние два абзаца имеют класс под названием “sub-paragraph”. Теперь создадим правило CSS для этого следующим образом:

.sub-paragraph {
   color:  red;
}
